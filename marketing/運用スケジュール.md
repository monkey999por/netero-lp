# Netero マーケ運用スケジュール（アリーナ週次ローテ）

アリーナ別ウェイトリストを回す運用のマスター。
ルート（`/`）は **リロードのたびにアリーナをランダムに切り替える**（直前と同じは避ける。`index.html`）。
さらに各LPには**浮遊するアリーナ・シャッフルBot**（右下）があり、クリックで別アリーナへ飛べる。
各アリーナLPは `/<slug>/` で**常時直アクセス可能**。X/IG からは狙ったアリーナのパスを直リンクする。

> マーケの「今週の推し」は運用側で決める（下記の週次リズム）。ルートのランダム切替は
> ドメイン直訪問者に全アリーナを見せて回遊させるための仕掛け。両者は独立。

### ルートの挙動
ルート（`/`）を開く／リロードするたびにランダムなアリーナへ飛ぶ（直前と同じは避ける、`localStorage` で記憶）。
「今週どのアリーナを推すか」はマーケ運用側の判断で、下記の週次リズムに沿って発信する（ルートのランダム切替とは独立）。

## 毎週の実行手順

1. **月曜**: 当週アリーナのデッキを開く。X/IG の各投稿の `《FORM_URL_...》` が本番URLに差し替わっているか確認（→ 下記チェックリスト）。
2. **毎日**: デッキの当日投稿を出す。**投稿時間は夜21:00〜22:30 が本命**（朝7-8時・昼12時も可）。X はリンクを本文に入れず1stリプ/プロフィールへ。
3. **画像**: 該当日の画像（`images/<arena>/*.png`）を添付。LP紹介日は当該LPのスクショを撮る。
4. **エンゲージ（最重要）**: 投稿して終わりにしない。**当週アリーナに関連するタグ（#今日の積み上げ / #勉強垢 / #個人開発 等）の他人の投稿に毎日5〜10件🔥リプ**。「気配」プロダクトの体現がそのまま最強の集客。
5. **日曜**: 当週アリーナの登録数・メール記入率を記録（下記KPI）。翌月曜、ルートが次アリーナへ自動で切り替わる。

## アリーナ別フォーム（要・事前準備）

テーマ別に登録数を比較するため、**アリーナごとに別々の Google フォーム**を用意し、各LPのプレースホルダを差し替える:

| アリーナ | LP内プレースホルダ | 状態 |
|---|---|---|
| AIで稼ぐ | （設定済み） | ✅ 稼働中 |
| 資格試験 | `《FORM_URL_shikaku》` | ⬜ フォーム作成 → 差し替え |
| 個人開発 | `《FORM_URL_indie》` | ⬜ フォーム作成 → 差し替え |
| 大学受験 | `《FORM_URL_juken》` | ⬜ フォーム作成 → 差し替え |

フォーム設問は ai-earn と同型（今何をしているか / 期待度1〜5 / 何を求めるか / メール任意 / X任意）。文言だけ各テーマに寄せる。

## 週次KPI（撤退ライン接続）

各アリーナ週の終わりに記録し、`docs/00-結論と戦略サマリ.md` の Phase 1 撤退ラインで判定:
- **登録数**（30日通算で30件未満 → そのアリーナは封印候補）
- **メール記入率**（50%未満 → 関心が弱い）
- 期待度スコア 4以上の割合 / 「他人の努力からモチベを得たい」選択率

→ **テーマ別に独立判定**。刺さったアリーナ（登録が伸びた順）から Phase 2（プライベートβ）を開く。1周（4週）回して各アリーナのベースライン、2周目で再現性を見る。

## 注意

- **大学受験は未成年ユーザが多い**。煽り・不安商法を避け「一緒に頑張る自習室」トーンを厳守（デッキに明記）。
- 数字（登録数など）は実数で出す。盛ると build-in-public の信頼が死ぬ。
- ルートは noindex。SEO を狙うのは各アリーナLP（個別に title/OG 設定済み）。
