# Netero — Waitlist LPs

> 全員のタスクが、公開で流れる。隠す場所はないから、続く。
> **Netero** は同じ目標を持つ人間のタスク・進捗・完了がすべて公開で流れる **タスクSNS**。
> 「焦り」を燃料に、全員の生産性を底上げする。

このリポジトリは Netero の**ウェイトリスト・ランディングページ**を管理する。
テーマ（＝アリーナ）ごとに独立した LP を持ち、**1ドメイン・パス違い**で並べる。

```
example.com/            → フラッグシップへリダイレクト
example.com/ai-earn/    → 「AIで稼ぐ」アリーナ ウェイトリスト
example.com/toeic/      → 「TOEIC」アリーナ（今後）
example.com/kintore/    → 「筋トレ」アリーナ（今後）
...
```

## コンセプト

- **タスクは、すべて公開** — 宣言・進捗・完了が全員に流れる。「見られている」は最強の締切
- **同じ山を登る奴しかいない** — アリーナはテーマ制。同じ目標の人間だけが集まる
- **週次リーグ** — 消化ペースが近い5〜10人で毎週リーグ戦。昇格か、降格か
- **リアクションは🔥だけ** — マウントも説教も勧誘も、構造的に不可能

## 構成

各アリーナ LP は **完全に独立した自己完結 HTML**（デザインはアリーナごとに自由に変える）。

```
/
├── index.html        # ルート。フラッグシップへリダイレクト
├── ai-earn/
│   └── index.html    # 「AIで稼ぐ」LP（1ファイル完結）
└── <slug>/
    └── index.html    # アリーナを足すときはフォルダを1個増やすだけ
```

## デプロイ

**Cloudflare Pages**（GitHub 連携）。`main` に push すると自動デプロイされる。
ビルド不要の静的サイト（Framework: None / Build command: なし / Output: `/`）。

## ローカルで見る

```bash
python3 -m http.server 8080
# → http://localhost:8080/ai-earn/
```

## 新しいアリーナ LP を足す / 提案する

新しいアリーナ（テーマ）でウェイトリストを開きたい人は、フォルダを1つ追加するだけ:

```bash
mkdir <slug>            # 例: toeic, kintore, blog
# <slug>/index.html を作る（既存LPをコピー改変 or ゼロから別デザイン）
```

- `<title>` / OG タグはそのアリーナ専用に書く
- ウェイトリストボタンの `href` はそのアリーナ専用の登録フォームにする
- デモUIのフェイクデータには「画面イメージ」表記を残す

Pull Request 歓迎。あなたのアリーナが、次のリーグの創設ラインになる。
