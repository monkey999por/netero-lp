## 何を作るのか

**Netero** = 「全員のタスクが公開で流れるタスクSNS」。同じ目標を持つ人間のタスク・進捗・完了が相互に見えることで、
**焦り・鼓舞・武者震い**を燃料に全員の生産性を底上げする。対戦アプリではなく「社会的可視化」プロダクト。

> オンライン自習室のタスク版。図書館で勉強が捗るのは監視されているからではなく、
> 周りが手を動かしている気配があるから。その気配をタスクデータで再現する。

## 結論（3行）

1. **アイデアの核（他人のタスク消化が見える武者震い）は筋が良い**。心理学的根拠（similar-others 効果）と、既存の習慣化アプリの弱点の裏返しの両面で成立する。
2. ただし**いきなりフルスクラッチで作らない**。まず Phase 0/1（LP + ウェイトリスト検証、実装ゼロ）で需要を測ってから実装に進む「フェーズゲート」方式。
3. クライアントは**モバイルファースト・local-first**。ソーシャル/ランキングだけを薄いサーバで持つ。

## プロダクト名とワンライナー

**Netero**

> 「AIで稼ぐ」を本気でやる人間だけのタスクSNS。
> 全員のタスク・進捗・完了が公開で流れる。隠す場所はないから、続く。

テーマアリーナ制（旗艦「AIで稼ぐ」→ TOEIC・筋トレ・資格・大学受験など何でも）。
同じ山を登る者同士の競争・消化量・出せた結果が相互に見えることで全員をヒートアップさせる。芯は「**焦りによる強制的な努力**」。

## 設計判断の原則（このプロダクトを通すために守ること）

| 原則 | Netero での扱い |
|---|---|
| 純ソフトウェアで在庫レス・自動化天井が高いこと | ✅ freemium 自動課金・運用自動化で「寝てても回る」を目標 |
| ペインが無料で自己解決できないか | ⚠️ 「タスク管理」自体は無料で自己解決可能。売るのは管理機能ではなく**継続の強制力（社会的コミットメント）**。人はパーソナルジム・コーチング・習慣化アプリ課金に「監視されること」への対価を既に払っている → Phase 1 の検証テーマ |
| 後発で速度・効率競争に入らない | ⚠️ 習慣化アプリの先行者は多い。正面勝負は負け筋。**「監視」ではなく「同じ空間で作業している気配」**にポジションをずらす（01） |
| 規約・法律の地雷を入口で潰す | ⚠️ 罰金プール機能（将来）は賭博・資金決済法の確認が必須 → v2 送り。個人データの相互可視化は「公開が前提」の建て付け + 規約で対応。MVP（可視化のみ）には地雷なし |
| 自分用で価値実証してから広げる | ✅ 作者自身が一次ターゲットのど真ん中。ドッグフーディングで実証しつつ、Phase 0/1 は「自分のデータを見せる」だけで需要を測れる |

## フェーズゲート戦略

| Phase | 内容 | ゲート（次に進む条件） |
|---|---|---|
| **Phase 0** 共有画像 export | 達成サマリの共有画像を生成し X に貼れるようにする（build-in-public の弾を兼ねる） | 自分が継続的にシェアしたくなるか |
| **Phase 1** LP + ウェイトリスト検証 | LP + ウェイトリスト + アンケート。実装ゼロ | ↓の撤退ライン参照 |
| **Phase 2** プライベートβ | connector 方式の最小構成（02）。友人・登録者 5〜20人 | 週次アクティブ率 50% 以上が4週続くか |
| **Phase 3** 公開 + 課金 | freemium 課金開始（03） | Phase 2 で「他人の気配で自分の消化数が増えた」定量実感 |

## 撤退ライン（着手前に数値で書く）

- **Phase 1**: ウェイトリスト公開から30日で **登録 30件未満 → 封印**。登録が30件以上でも、熱量シグナルが弱ければ再検討:
  - **メール記入率**（＝実際に招待できる母数）が **50% 未満**なら「関心はあるが弱い」
  - 期待度スコア（1〜5）の **4以上が 40% 未満**、または「他人の努力からモチベを得たい」を選んだ人が過半未満ならコンセプト訴求が刺さっていない
  - ※ 価格の妥当性検証は Phase 2 のβ内（実使用後の課金オファー）に後ろ倒し
- **Phase 2**: β開始から8週で **週次アクティブ率 50% 未満 → 封印**。可視化前比で自分の消化数が増えていなければコンセプト根拠を棄却。ここで初めて価格検証（課金オファー転換率が実ユーザの5%に届くか）。
- **共通**: 本業（主力の収益事業）の稼働をこの件が2週連続で食ったら、フェーズ問わず即中断して本業に戻る。

## 主要な設計決定ログ

- **名称 Netero に決定**
- **タスクSNS化**: タスクのタイトル・内容・進行/完了がそのまま全員に公開で流れる。非公開オプションは持たない（「見せる前提のタスク管理」こそがプロダクト）
- **テーマアリーナ制**: 「AIで稼ぐ」を旗艦に、テーマを順次開設。ウェイトリストもテーマ別に複数展開
- **クライアントはモバイルファースト・local-first**、PWA → Expo の2段階配布、匿名ファースト onboarding
- **リーグ内順位表 + 昇格/降格制（Duolingo 型）**。全体グローバルランキングは出さない
- **公式 X での表彰 bot + MVP 投票**（投票は X ネイティブポールで実装ゼロ、掲載は opt-in 限定）
- **PWA のデータは IndexedDB（+Dexie）**、streak 計算は依存ゼロの純関数として実装
