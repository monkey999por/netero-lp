# Netero — プロダクト設計

> 設計ドキュメント 01 / 前提: [00 結論と戦略サマリ](00-結論と戦略サマリ.md)

## コア体験の定義

**対戦ではない。社会的可視化による武者震いである。**

狙い:
> 継続している人たちが他の人のタスク消化を見て、モチベーションを上げて競争を活性化させる。
> 使っている人全体の生産性を底上げする（焦りや、他の人もやっているという状況での鼓舞、武者震い）。

一言にすると「**オンライン自習室のタスク版**」。図書館で勉強が捗るのは監視されているからではなく、
**周りが手を動かしている気配があるから**。その気配をタスク消化データで再現する。

### 芯にある3つの上書き

1. **テーマアリーナ制**: ユーザはまず大枠テーマ（アリーナ）に所属する。旗艦は「**AIで稼ぐ**」。他に TOEIC・大学受験・資格・筋トレなど何でも設定可能。その中で各自が**独自にタスクを作って**こなす（やり方の自由＝攻め方の個性が見えるのもコンテンツ）。similar-others 効果（下記）はテーマ一致で最大化される — 「同じ山を登っている人の足音」。新規アリーナは人数閾値（例: 待機5人）で順次開設し過疎を防ぐ。
2. **タスクSNS**: タスクのタイトル・内容・進行/完了が**そのまま全員に公開で流れる**。非公開オプションは持たない（X に書くのと同じ覚悟で書く場所、と登録時に明示）。Strava がランニングでやったことをタスクでやるイメージ。焦りの解像度が「誰かが3件消化した」から「**誰かが『○○を納品』を完了した**」に上がり、競争・消化量・出せた結果の3層すべてが中身込みで燃料になる。
3. **芯は「焦りによる強制的な努力」**。ただし焦りと恥辱は区別する: 焦りは「動いている他人の気配」（フィード・アリーナ統計）から生ませる。「お前は遅れている」という名指し・停滞者の晒しはやらない（離脱製造機）。

### 心理学的根拠

- **他人の社会規範の模倣で行動力が上がる。真似する相手は自分と状況が似ている人がいい** → リーグ（同レベル帯マッチング）の直接の根拠。トップランカーを見せても凹むだけで、「自分と似た人が今日もやってる」が効く。
- **社会的促進（social facilitation）**: 単純作業は他者の存在で速くなる。
- **コミットメントデバイス**: 宣言・可視化は実行率を上げる。

### 競合の弱点から導く設計原則

既存の習慣化アプリ（証明写真ベースの相互監視型など）の弱点を裏返す:

| 既存アプリの弱点 | Netero の設計原則 |
|---|---|
| 「同じ志のプレッシャー」で自分との戦いでなくなる | 証明写真の提出義務を作らない。done は自己申告 + 自動連携。監視アプリではなく気配アプリ |
| 疎外感があると続かない。所属意識が要る | リーグは小さく（5〜10人）・週次リセット・脱落者を晒さない。「今週も同じ面子」の緩い所属感 |
| 全員が同じことをするグループは飽きる | v2 で**非対称ロール**を導入（後述） |
| 属性（性別・年齢）の可視が合う人と合わない人がいる | ハンドル制（実名・性別・年齢は持たない）。見えるのは「何をやってる人か」だけ |

加えて設計の北極星: **「アクセスする手順を減らすだけで大きな価値になる。機能を足すより手順を消す」**。
→ done の記録は **1タップ or 自動連携**。記録の手間が1動作でも増えるアプリは続かない。同じ理由で**スマホ完結**が必須。

## 機能セット

### MVP（Phase 2 プライベートβ）

1. **草グラフ + streak プロフィール** — 日別 done 数・現在/最長 streak。プロフィールに全タスク（進行中/完了）も並ぶ。
2. **リーグ（週次・同レベル帯・5〜10人、昇格/降格制）** — 同一アリーナ内で消化ペースが近い者を自動編成。**リーグ内に限り順位表を出す**（Duolingo 型: 同レベル帯の小集団内なら順位は凹みでなく火種）。Bronze → Silver → Gold → Diamond のタイア制。**全体グローバルランキングは出さない**。
3. **Live フィード（タスクがそのまま流れる）** — 「たった今、◯◯が『Kindle原稿 3章』を完了」。宣言・進行・完了の3イベント。開いた瞬間「動いてる人がいる」と中身込みで分かる = 武者震いの発生装置。アリーナ内タスク一覧ビューも持つ。
4. **応援リアクション（🔥 のみ）** — コメント機能は作らない。
5. **殿堂 opt-in（X 表彰への同意フラグ）** — 週間トップ・改善率トップ・streak 節目を公式 X が自動投稿で表彰。掲載は opt-in のみ。MVP 投票は X ネイティブポールで（実装ゼロ・治安リスクをアプリ外に隔離）。

### v2（Phase 3、検証通過後）

6. **同時刻もくもくスプリント（唯一の"対戦"要素）** — 参加表明 → 同時刻に各自作業 → 消化数が並ぶ。対戦は同時性のある協調型に限定。
7. **リーグ内ロール（人狼型・非対称）** — 週替わりで「ペースメーカー」「見届け人」等の軽いロール。
8. **罰金プール（コミットメントデバイス）** — 法務確認が通るまで実装しない。

### 作らないもの（明示的スコープ外）

- チャット・コメント欄・DM（治安コスト。勧誘・商材屋の営業導線を作らない）
- テーマ横断のグローバルランキング（アリーナが違えば比較に意味がない）
- タスクへの添削・アドバイス（監視アプリ化・マウント発生を防ぐ）
- 証明写真・エビデンス提出
- フィード内 URL のリンク化（スパム対策。テキストは可、飛べない）
- ゲーム的レベル・ガチャ（ソロ運用ではコンテンツ供給で息切れする）

## ターゲットユーザ

1. **一次: 個人開発者・副業勢・資格勉強勢**。「毎日積み上げたいが一人だと途切れる」層。X の #今日の積み上げ タグに毎日数千件投稿 = 需要の既存証拠。
2. **二次: 既存の習慣化アプリの不満層**（証明写真がだるい・グループが重い）。

## 成功指標

- 北極星: **「入会後、ユーザの週次消化数が入会前より増えているか」**（生産性の底上げがコンセプトなので DAU より消化数の増分）
- 補助: 週次アクティブ率 / streak 継続率 / フィード閲覧 → 1時間以内の done 発生率（気配→行動の転換率）
